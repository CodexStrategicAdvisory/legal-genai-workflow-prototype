## How It Works

This prototype simulates a no‑code Legal GenAI workflow by breaking a contract‑review task into structured, deterministic steps. Instead of calling an LLM, the script uses patterns from the Legal GenAI Component Library to demonstrate how an AI Practitioner would design a safe, auditable workflow inside the LLM Suite.

### 1. Load Input
The workflow begins by loading a sample contract from `sample_contract.txt`, representing a high‑volume legal document type.

### 2. Two‑Pass Summarization
The script applies a simplified version of the [two‑pass summarization](ca://s?q=Explain_two_pass_summarization) pattern:
- Pass 1 extracts key facts and obligations.
- Pass 2 rewrites them into a concise, plain‑language summary.

### 3. Schema‑Driven Output
The workflow formats results using a predefined JSON schema, mirroring how structured outputs are enforced in the LLM Suite. This ensures:
- Consistency  
- Deterministic fields  
- Easy downstream consumption  

### 4. Citation‑or‑Abstain Logic
The script demonstrates the [citation‑or‑abstain](ca://s?q=Explain_citation_or_abstain) pattern by only returning extracted terms when they appear explicitly in the source text.

### 5. Risk‑Aware Extraction
The workflow identifies:
- Payment terms  
- Confidentiality obligations  
- Key duties  

This mirrors how risk signals or obligations would be extracted in a production workflow.

### 6. Deterministic Output Rendering
Finally, the script prints a structured summary to the console, simulating the type of output a no‑code workflow would return to an attorney or downstream system.
